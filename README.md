# Clicky

Clicky is a macOS menu bar companion that lets you talk to an AI assistant while you work. It lives in the status bar, listens when you hold `control + option`, captures the current screen context, and responds with both voice and an on-screen cursor that can point at relevant UI elements.

## Project Description

Clicky is designed to feel less like a chat window and more like a helpful presence on your desktop. Instead of forcing the user to switch contexts, open a separate app, or type long prompts, Clicky stays available in the menu bar and uses a push-to-talk flow.

When the user speaks, Clicky:

- records microphone audio
- transcribes the speech
- captures the relevant screen context
- sends the transcript and screenshots through a Cloudflare Worker proxy to OpenAI
- streams back a response
- speaks the answer aloud
- optionally points at a UI element with Annie, the on-screen cursor companion

The macOS app never ships raw API keys. All model, transcription, and text-to-speech requests go through the worker.

## Problem Statement

Most AI assistants still live in a separate interface. That creates friction.

If a user is trying to understand what is on screen, find a button, navigate an unfamiliar app, or get quick help while coding, they usually need to:

- switch away from what they are doing
- describe the screen manually
- paste screenshots into a chat
- interpret a text-only answer on their own

This breaks flow and makes the assistant feel disconnected from the actual task.

The problem Clicky solves is: how do you make an AI assistant feel present, screen-aware, and fast enough to help in the middle of real desktop work without turning it into a full-screen app?

## Solution

Clicky solves this by combining four ideas:

1. A menu bar-only app

The assistant is always available without taking over the desktop. There is no dock icon and no main window.

2. Push-to-talk voice input

The user holds `control + option`, speaks naturally, and releases to send. This keeps interaction fast and low-friction.

3. Screen-aware reasoning

Clicky captures the screen at the moment of the request, builds structured context around the active app and cursor area, and sends that context to the model so the assistant can answer based on what the user is actually looking at.

4. Visual pointing

Annie can point at things on screen using `[POINT:x,y:label:screenN]` tags generated by the model. The app maps those coordinates back onto the correct monitor and animates the cursor to the target.

## How It Works

### High-Level Flow

`hotkey -> mic capture -> transcription -> screen capture -> OpenAI chat -> point parsing -> TTS -> cursor pointing`

### Step-by-Step

1. The app runs as a menu bar utility.

`leanring_buddyApp.swift` starts the app, creates the panel manager, and starts `CompanionManager`.

2. The user presses `control + option`.

`GlobalPushToTalkShortcutMonitor.swift` listens system-wide for the shortcut, and `BuddyDictationManager.swift` begins recording audio.

3. The user releases the shortcut.

The active transcription provider finalizes the transcript. By default, this is OpenAI transcription through the worker proxy.

4. Clicky captures screen context.

`CompanionScreenCaptureUtility.swift` captures the display state, identifies which screen contains the cursor, builds a cursor-centered crop, and prepares prioritized screenshots for the model.

5. Clicky sends the request to OpenAI.

`CompanionManager.swift` combines:

- the user transcript
- recent conversation history
- structured screen summary
- ordered screenshots

and sends them to `OpenAIChatCompletionsClient.swift`, which calls the worker’s `/chat` route.

6. The model responds with speech text and, when useful, a point tag.

If the response includes something like `[POINT:840,312:save button]`, Clicky parses it and turns it into a real screen coordinate. A second-pass refinement crop can be used to improve pointing accuracy before the cursor moves.

7. Annie speaks and points.

`OpenAITextToSpeechClient.swift` calls the worker’s `/tts` route and plays the response aloud. `OverlayWindow.swift` handles the floating cursor overlay, waveform, spinner, and the pointing animation across monitors.

## Core Components

- `CompanionManager.swift`: the central app state machine
- `BuddyDictationManager.swift`: push-to-talk recording and transcript lifecycle
- `CompanionScreenCaptureUtility.swift`: screenshot capture and screen-understanding context
- `OpenAIChatCompletionsClient.swift`: streaming chat requests
- `OpenAITextToSpeechClient.swift`: text-to-speech playback
- `OverlayWindow.swift`: Annie’s overlay, voice states, and pointing animation
- `worker/src/index.ts`: Cloudflare Worker proxy for OpenAI and AssemblyAI

## Why The Worker Exists

The worker keeps secrets out of the macOS app and provides a single controlled boundary for:

- chat completions
- speech transcription
- text-to-speech
- temporary AssemblyAI token generation

That means the shipped app can stay lightweight and safer, while all third-party API access stays server-side.

## Development Notes

- Open the app in Xcode with `open leanring-buddy.xcodeproj`
- Run the `leanring-buddy` scheme from Xcode
- Do not run `xcodebuild` from the terminal for this project, because it can invalidate macOS TCC permissions such as Screen Recording, Accessibility, and Microphone access

## Current Experience

Today, Clicky supports:

- voice-first interaction from anywhere on macOS
- screen-aware assistance with multi-monitor support
- spoken responses
- UI element pointing with Annie
- screenshot command handling
- onboarding and menu bar controls

The goal is to make desktop AI assistance feel ambient, fast, and grounded in what the user is doing right now.
